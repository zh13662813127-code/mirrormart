# 镜市 MirrorMart — 产品需求文档 (PRD)

> 版本 1.0 · 2026-03 · 状态: Phase 0 准备中

---

## 1. 产品概述

### 1.1 一句话定义
镜市是一个开源的中国社会模拟引擎，用大量 LLM 驱动的虚拟个体在模拟的中国互联网平台中互动，通过蒙特卡洛多路径推演输出概率分布结果。

### 1.2 核心价值主张
- **不是一个答案，而是一千种可能。** 输出概率地图而非确定性回答。
- **中国互联网环境的真实模拟。** 虚拟消费者在模拟的小红书种草、淘宝比价、抖音被打动、微博讨论。
- **随时改变条件，看世界怎么变。** 注入变量 → 联网校准 → 重跑多路径。
- **不只电商。** 舆情推演、内容传播、商业决策均可。
- **开源，人人可扩展。** Apache 2.0，社区驱动。

### 1.3 与镜鱼 MiroFish 的关系
镜鱼是通用群体智能引擎（Twitter + Reddit，全球场景）。镜市是同一核心理念的中国本土化实现，新增三大能力：中国平台环境、蒙特卡洛多路径、联网搜索校准。

## 2. 目标用户

### 2.1 主要用户
| 用户角色 | 痛点 | 镜市如何解决 |
|----------|------|-------------|
| 电商品牌/运营 | 投放前无法预判效果，拍脑袋决策 | 多路径模拟不同策略的概率分布 |
| 公关/舆情团队 | 危机爆发后才能响应，无法预演 | 模拟舆情传播路径和临界点 |
| 创业者/产品经理 | 无法低成本验证市场假设 | 虚拟消费者测试产品/定价/营销方案 |
| 社科研究者 | 社会实验成本高、周期长 | 可重复、可控的社会行为模拟环境 |

### 2.2 次要用户
- 开源社区贡献者（新增平台插件、Agent 模板、场景包）
- 教育场景（消费者行为、社会学教学辅助）

## 3. 功能需求

### 3.1 核心四步体验

#### Step 1: 设定（Setup）
- **输入**: 用户用自然语言或结构化表单描述起点
  - 具体型: "我有一款面膜，定价 59 元，主打氨基酸温和配方，想在小红书推"
  - 开放型: "如果某奶茶品牌被曝食安问题，舆情怎么演变"
- **系统行为**:
  - 解析用户输入，确定场景类型（电商/舆情/内容传播/自定义）
  - 自动生成虚拟个体群（默认 100-200 人）
  - 每人拥有：人格画像 + 社会关系 + 平台使用习惯 + 决策偏好
  - 构建对应的模拟平台环境

#### Step 2: 模拟（Simulate）
- 同一起点运行多次（默认 5-10 次，可增至 50-100 次）
- 每次模拟中 Agent 在平台环境中自由行动
- 因行为随机性产生不同路径
- 结果汇总为概率分布

#### Step 3: 观察（Observe）
- **宏观统计视图**: 概率分布图，各结果类型出现频率
- **群体动态视图**: 传播网络、观点变化趋势、关键转折点
- **个体旅程视图**（核心差异化）: 可选择任一 Agent 查看完整行为链
  - 示例: 成分党在小红书刷到产品 → 看评论 → 淘宝比价 → 犹豫 → 看到朋友推荐 → 购买

#### Step 4: 干预（Intervene）
- 修改任意条件后重跑多路径模拟
- 干预类型: 改价格、注入差评、竞品活动、增加 KOL、改投放策略
- 系统自动联网搜索相关真实数据校准参数
- 核心价值: 同一群体在条件 A vs 条件 B 下的概率分布对比

### 3.2 平台环境模拟

每个平台是一组简化规则，不是复现真实算法：

| 平台 | 核心模拟规则 | 支持的用户行为 |
|------|-------------|---------------|
| 小红书 | 种草发现流 + 搜索 + 评论互动 | 发笔记、点赞、收藏、评论、搜索 |
| 抖音 | 短视频互动流 + 完播率加权推荐 | 观看、点赞、评论、转发、搜索 |
| 淘宝 | 搜索比价 + 评价体系 + 销量排序 | 搜索、浏览、比价、加购、购买、评价 |
| 微博 | 热搜机制 + 转发链扩散 | 发帖、转发、评论、搜索热搜 |

**Phase 0 仅实现: 小红书信息流 + 淘宝搜索比价**

### 3.3 虚拟个体（Agent）

#### 人设层
- 基础属性: 年龄、性别、城市、收入、职业
- 消费画像: 品类偏好、价格敏感度、决策风格（冲动/理性/从众）
- 性格特征: 大五人格维度
- 社交属性: 影响力等级、社交活跃度

#### 平台行为层
- 每平台的使用频率和时段
- 内容消费偏好（图文/视频/评测）
- 互动倾向（高互动/潜水/创作者）
- 信息处理方式（成分党/颜值党/价格党）

### 3.4 联网搜索校准
- 触发时机: 每次干预后、模拟初始化时
- 搜索内容: 同品类价格带、竞品数据、历史类似事件传播数据
- 作用方式: 注入到 Agent 的认知参数中，不直接决定结果
- 数据源: 搜索引擎 API + 中国平台数据接口

## 4. 非功能需求

### 4.1 性能
- Phase 0: 20 Agent × 5 次路径 在 5 分钟内完成
- Phase 1: 200 Agent × 10 次路径 在 30 分钟内完成
- Phase 3: 1000 Agent × 50 次路径 在 2 小时内完成

### 4.2 成本
- Phase 0 单次完整实验: < $5（API 调用）或接近 $0（本地模型）
- 长期目标: 单次 1000 Agent 模拟 < $5

### 4.3 可扩展性
- 新平台可通过插件添加，无需修改核心引擎
- 新场景可通过 YAML 配置，无需写代码
- Agent 画像模板可由社区贡献

### 4.4 开源要求
- 许可证: Apache 2.0
- 文档: 中英双语
- CI/CD: GitHub Actions
- 包管理: PyPI 发布

---

## 5. 不做什么（明确边界）

- **不做** 真实平台算法的精确复现
- **不做** 绝对数值预测（如"你的产品会获得 10 万曝光"）
- **不做** 真实用户数据采集或隐私相关功能
- **不做** 实时对接真实平台 API 进行操作
- **不输出** 确定性结论，只输出概率分布和相对比较

## 6. 成功指标

### Phase 0 (3 天)
- [ ] 简化小红书环境能驱动 Agent 产生有意义的互动链路
- [ ] 5 次模拟结果有足够差异，概率分布有意义
- [ ] 至少 1 个 Agent 的个体旅程是连贯、可信的

### Phase 1 (1 月)
- [ ] 25 Agent 在小红书环境中持续运行不崩溃
- [ ] Agent 行为日志可读、可分析
- [ ] 有基本的可视化输出

### v1.0
- [ ] GitHub 500+ Stars
- [ ] 至少 3 个社区贡献的 Agent 模板或场景包
- [ ] 1 个可验证的电商模拟案例（与真实数据对比方向一致）

## 7. 术语表

| 术语 | 定义 |
|------|------|
| Agent | LLM 驱动的虚拟个体，拥有独立人格和行为逻辑 |
| 平台环境 | 模拟的中国互联网平台，由简化规则集定义 |
| 路径 (Path) | 一次完整模拟从起点到结束的全过程 |
| 分支 (Branch) | 蒙特卡洛模拟中同一起点的不同运行实例 |
| 干预 (Intervention) | 在模拟中注入新变量或改变条件 |
| 校准 (Calibration) | 用真实数据调整模拟参数 |
| 涌现 (Emergence) | Agent 互动中自发产生的群体行为模式 |
| 种草 | 小红书语境下通过内容推荐促成购买意向的行为 |

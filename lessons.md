## 设计阶段经验

### 1. 平台环境不需要复现真实算法
**背景**: 最初担心不了解小红书/淘宝的推荐算法会导致模拟无意义。
**结论**: OASIS 论文已证明，简化规则 + 真实Agent人设 = 有意义的涌现行为。环境规则不需要完美，Agent 的行为逻辑才是关键。
**行动**: 每个平台只定义 5-10 条核心规则，不追求算法精确。

### 2. 模拟的价值在于相对比较
**背景**: 担心模拟结果的绝对数值不准确。
**结论**: 不需要准确。只要两个方案在同一环境下比较，相对差异就有参考价值。就像天气预报不完全准，但你还是看了再决定带不带伞。
**行动**: 永远不输出绝对数值预测，只输出概率分布和相对比较。

### 3. 竞品都没做中国市场
**背景**: 竞品调研发现 Simile ($100M融资) 和 Artificial Societies (YC W25) 验证了市场需求。
**结论**: 两者都没有中国平台支持。中国互联网生态（种草文化、短视频算法、比价习惯、热搜机制）无法简单翻译移植。
**行动**: 中国平台环境和中国原生LLM支持是镜市的核心差异化。

### 4. Phase 0 必须极简
**背景**: 技术方案涉及 Redis/Neo4j/TimescaleDB/Ray 等大量基础设施。
**结论**: Phase 0 的目的是验证核心假设，不是构建完整系统。过早引入基础设施会拖慢验证速度。
**行动**: Phase 0 只用 Python + JSON + 一个LLM API。3天内必须出结果。

## 技术经验 (编码开始后持续更新)

### 5. MiniMax M2.5 推理模型输出为空
**背景**: call_llm 返回内容为空，模拟 fallback 到"走神了"
**原因**: M2.5 是推理模型，先输出 `<think>...</think>` 占用大量 token，max_tokens=512 全被思考用完
**解决**: 剥离 `<think>` 标签 + 调大 max_tokens 到 2048
**行动**: 接入推理模型时默认 max_tokens >= 2048

### 6. 成分党卡在无限 view 循环
**背景**: ingredient_nerd 在 Branch 0 连续 view 同一商品 14 次，从未购买
**原因**: 商品描述中没有详细成分表，Agent 每次查看后仍找不到"成分表符合预期"的触发条件
**解决**: 淘宝商品对象增加 `ingredient_list` 字段；Agent 记忆中追踪同一商品查看次数，超过3次强制做最终决策
**行动**: Phase 1 在 decide 提示词加"如果已查看同一商品超过3次，必须做买或不买的最终决定"

### 7. 随机种子对网络密度影响过大导致分支极化
**背景**: Branch 0/1 有大量购买(15/10次)，Branch 2/3/4 购买为0，XHS 点赞数恒为初始值5
**原因**: init_following 的随机种子影响了关注网络密度。低密度网络下内容无法传播，Agent 互相看不到行为
**解决**: init_following 增加 min_follows_per_agent 参数（默认3），确保每人至少关注3个人
**行动**: Phase 1 修复网络初始化，同时增加"热门内容随机曝光"机制（不依赖关注关系）

### 8. 价格猎手和理性买家转化率为零
**背景**: price_hunter 和 rational_buyer 大量搜索、加购物车，但从未购买
**原因**: 竞品同时出现在搜索结果中（¥39/¥59/¥89），触发持续比较但无法收敛决策
**解决**: 在 decide 提示词中加入"如果已经查看该商品超过3次，必须做最终决策"的强制规则
**行动**: Phase 1 给 Agent 增加购买意向阈值触发机制：purchase_intent > 0.7 时强制进入决策流程
